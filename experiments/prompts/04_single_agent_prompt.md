# 실험 2-1: Single Agent

정상 `wishlist-court` MCP 서버를 사용하세요.

당신은 단일 에이전트입니다. 계획, 데이터 조회, 판정, 보고서 작성을 모두 직접 수행하세요.

작업:

1. 위시리스트 전체를 읽으세요.
2. 각 항목의 보유품 중복 여부와 예산 위험을 확인하세요.
3. 각 항목의 충동구매 위험을 판단하세요.
4. 최종 판결을 내리세요.
5. 결과를 `single_agent_verdict.md`로 저장하세요.

측정할 항목:

- 전체 소요 시간
- MCP 호출 수
- 토큰 사용량
- 누락된 판단 기준
- 결과의 일관성

목표:

- 이 작업에서 Single Agent가 가장 토큰 효율적인지 확인한다.
